Graph Generation & Task Adaptation Strategy

This document details the macro-level structural topology of the Dynamic DAG Neural Network. It explains the algorithmic generation of layers and nodes, the aggregation of signals into the final Fully Connected (FC) layer, and the systemic modularity that allows this single architecture to adapt to varying input types and distinct machine learning problems.

1. Algorithmic Formation of Layers and Nodes

Unlike traditional networks where layer widths (e.g., 512 neurons, 256 neurons) are manually hardcoded by the designer, the hidden structure of this network is generated procedurally using statistical distributions. This creates a randomized, yet highly constrained, Directed Acyclic Graph (DAG).

1.1. The Pre-Compilation Generation Phase

Because the network will run on XLA (JAX's compiler), the dynamic graph cannot be generated on the fly during training. The entire physical topology—every node and every connection—is generated once during the model's initialization phase. Once initialized, the topology is frozen into static adjacency matrices and index arrays.

1.2. The Layer-by-Layer Sizing Logic

The network generates itself sequentially, layer by layer, starting from the Encoders.
The number of Decoder nodes instantiated in any new layer $L$ is strictly governed by the total connectivity requirements generated by layer $L-1$.

Determining Fan-Out: For every node $i$ in layer $L-1$, the algorithm samples a random integer from a specific Gaussian distribution. This integer represents how many downstream connections that specific node will output.

First Half of Network: Samples from a Gaussian with $\mu = 18$, strictly bounded between 8 and 24.

Second Half of Network: Samples from a Gaussian with $\mu = 12$, strictly bounded between 6 and 20.

Summing the Output Pool: The total number of required connections to the next layer is the sum of all fan-outs from layer $L-1$.

Calculating Node Count for Layer $L$: Because every Decoder node takes exactly 16 inputs, the approximate number of new Decoder nodes required in layer $L$ is calculated by dividing the total output connection pool from layer $L-1$ by 16.

Wiring the Connections: The generated output connections from $L-1$ are randomly shuffled and assigned to the input slots of the newly instantiated Decoder nodes in layer $L$.

Acyclicity Guarantee: Because connections are strictly drawn from layer $L-1$ (or earlier) and terminate only in layer $L$ (or later), it is mathematically impossible for a cyclic loop to form. The graph remains a strict DAG.

2. The Input Adapters (Stacked Encoders)

The entry point of the network consists of dedicated, fixed sets of Input Encoders.
Based on the architectural constraints, Encoders are not single, shallow transformations. They are designed as 2-layer stacked blocks.

Expansion: An individual input (e.g., an $n \times n$ feature map) is processed through these two stacked encoder layers, expanding significantly.

The 64-Output Manifold: Through the pairwise matrix multiplication mechanics described in the core specification, the stacked encoder block ultimately produces a dense manifold of 64 distinct $n \times n$ feature maps (often conceptually grouped as an $8 \times 8$ grid of channels).

Identity Preservation: These 64 outputs are then pushed into the dynamically generated Decoder graph. If an input type is not supplied by the user/dataset, its specific stacked encoder is deactivated, bypassing computation and outputting zero-tensors.

3. The Terminal Stage: The Fully Connected (FC) Layer

The internal Decoder graph is a chaotic, randomized routing system. To produce a useful, deterministic prediction, the network requires a structured terminal stage.

3.1. Gathering the "Remnants"

In standard networks, only the absolute final layer feeds into the output head. In this DAG architecture, the routing randomness means that some Decoder outputs from intermediate layers might never be assigned a connection to a downstream Decoder.

To ensure no computed information is lost, the final stage performs a massive gathering operation:

It collects the outputs of all nodes in the final physical layer of the graph.

It scans backwards through the network and collects the outputs of any Decoder node from previous layers that was left "unconnected" or untouched by the downstream routing algorithm.

3.2. Concatenation and Projection

The Concatenation Block: All gathered output tensors (each originally shaped $4 \times n \times n$) are flattened spatially and concatenated along their feature dimension. This forms a massive, single 1D vector representing the entire mixed cross-context state of the active graph.

The Output Head: This massive vector is passed through 1 or 2 Fully Connected (Dense) layers. These layers map the high-dimensional feature vector down to the specific dimensionality required by the problem (e.g., 10 logits for classification, 3 coordinates for bounding boxes).

4. Problem Adaptation Strategy

The true power of this architecture lies in its modularity. You do not need to redesign the internal DAG to solve different types of machine learning problems. The network adapts via its extremities (the Encoders and the FC Layers) and its dynamic routing.

4.1. Adapting to Different Input Modalities

The network handles multimodal or heterogeneous data through Encoder Modularity.

You instantiate a fixed number of Encoders, each assigned to a specific data modality (e.g., Encoder A for visual features, Encoder B for text embeddings, Encoder C for continuous sensor data).

Sparse Activation: If a specific problem or data sample only contains visual data, only Encoder A fires. Encoders B and C output their identity/zero states. The internal Decoder graph dynamically reacts: pathways heavily reliant on B and C will fail to meet their 12/16 activation threshold and will shut down. The network organically carves out a sub-graph optimized solely for visual processing.

4.2. Adapting to Different Output Formats

The network adapts to different predictive tasks through Head Swapping.

The complex internal routing and cross-context building remains identical whether you are predicting stock prices or classifying images.

To adapt the network to a new problem, you simply swap the final 1 or 2 Fully Connected layers.

Example - Classification: The FC layer is sized to [total_gathered_features, num_classes] and capped with a Softmax activation.

Example - Multi-Agent RL: The final stage might be split into two separate FC layers: one calculating a Policy distribution (actor) and one calculating a scalar Value estimation (critic).

4.3. Universal Training Paradigm

Because the internal graph is randomized and uses the Top-K determinant selection, the network acts as an immense, untrained hash function of features initially.

During training via backpropagation (using jax.grad), the network learns which of the 64 encoder outputs are most useful.

The gradients naturally flow back only through the active, Top-8 routing paths.

To adapt the network to a highly complex problem, you merely increase the initialization boundaries (e.g., higher $\mu$ for Gaussian connections, triggering a wider graph) and allow the backpropagation to find the optimal sub-routings.
